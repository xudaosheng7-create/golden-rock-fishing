# 游钓天下 — 广告系统设计方案

## 一、整体架构

```
┌─────────────────────────────────────────┐
│              微信小程序原生壳              │
│  wx.createRewardedVideoAd()              │
│  wx.createBannerAd()                     │
│  wx.createInterstitialAd()               │
│         ↕ postMessage                    │
├─────────────────────────────────────────┤
│          web-view: H5 游戏                │
│  ┌─────────────────────────────────┐    │
│  │        AdManager (单例)          │    │
│  │  ├─ showRewarded(scene, cb)     │    │
│  │  ├─ showBanner(scene)           │    │
│  │  ├─ hideBanner()                │    │
│  │  └─ showInterstitial(scene)     │    │
│  └─────────────────────────────────┘    │
│         ↓ 各游戏模块调用                  │
│  FishingControls / QuestPanel /          │
│  TankPanel / EquipPanel / ...           │
└─────────────────────────────────────────┘
```

**通信机制**：
- H5 → 原生：`wx.miniProgram.postMessage({ data })`
- 原生 → H5：URL 参数 或 JSSDK `window.postMessage`

**浏览器开发模式**：`confirm()` 弹窗模拟广告

## 二、广告位设计

### 2.1 激励视频（核心收入）

用户主动选择观看 15-30 秒视频，看完获得奖励。

| 场景 | 入口位置 | 奖励 | 冷却 | 日上限 |
|------|----------|------|------|--------|
| 🪱 **免费鱼饵** | 抛竿按钮旁，鱼饵=0时弹出 | 沙蚕 ×10 | 60s | 10次 |
| 🔧 **免费修理** | 装备损坏警告条 | 200金币（等价修理费） | 120s | 5次 |
| ✨ **3倍卖鱼** | 钓到鱼→「直接出售」弹窗 | 单条售价×3 | 无 | 3次 |
| 📦 **2倍批量卖** | 背包→鱼护→批量出售 | 整批售价×2 | 无 | 2次 |
| 📅 **补签** | 签到面板 | 补签 1 天 | 无 | 1次 |
| 🐟 **鱼缸加速** | 鱼缸面板 | 立即获得 2 小时产出 | 60s | 5次 |
| 🆘 **断线复活** | 搏鱼断线瞬间 | 恢复 50% 鱼体力 + 线张力清零 | 无 | 3次 |

### 2.2 底部横幅 Banner

常驻底部，不打断游戏。位置在底部菜单栏和操作按钮之间。

| 场景 | 触发条件 | 尺寸 |
|------|----------|------|
| 待机横幅 | idle 状态 > 10 秒 | 300×50 或 320×50 |
| 等待横幅 | waiting 状态 | 同上 |

Banner 不遮挡抛竿按钮（按钮在右下角，Banner 在底部中央偏左）。

### 2.3 插屏广告

全屏弹出，自然中断点展示。

| 场景 | 触发条件 | 概率 | 冷却 |
|------|----------|------|------|
| 切换钓场 | 点确认切换后 | 40% | 3 分钟 |
| 稀有鱼获 | 钓到 ★★★ 以上鱼 | 30% | 2 分钟 |
| 退出面板 | 关闭商城/背包面板 | 15% | 5 分钟 |

## 三、奖励平衡

### 原则

- 看广告是"便利加速"，不是"唯一途径"
- 所有奖励都有免费获取方式（花时间/花金币）
- 日上限防止刷广告破坏经济

| 广告行为 | 等价金币 | 免费替代 |
|----------|----------|----------|
| 领 10 鱼饵 | ~100g | 商城购买 |
| 免费修理 | ~200g | 花金币修理 |
| 双倍卖鱼 | 变值 | 正常卖 |
| 补签 | ~500g | 花钻石补签 |
| 鱼缸加速 | ~100g | 等 2 小时 |
| 断线复活 | 无法估价 | 接受损失 |

### 每日最大收益（看满所有广告）

```
10次鱼饵    = 1000g 等价
5次修理     = 1000g 等价
3次3倍单卖  = ~1500g 额外
2次2倍批量  = ~2000g 额外
1次补签     = 500g 等价
5次鱼缸     = 500g 等价
3次复活     = 无价（救命）
─────────────────────
合计约 6500g 等价 + 情感价值
```

**3倍 vs 2倍的设计意图**：
- 单条 3 倍 → 鼓励钓到鱼当场决策，增加即时爽感
- 批量 2 倍 → 鱼护囤货后统一卖出，倍率低但总量大

对比：正常钓鱼每日收入 ~2000-5000g。广告收益合理，不会让非广告玩家失衡。

## 四、微信小程序对接

### 4.1 原生壳需要做的事

在微信小程序项目（非 H5）中：

```javascript
// pages/game/game.js
Page({
  onLoad() {
    // 1. 创建激励视频广告
    this.rewardedAd = wx.createRewardedVideoAd({
      adUnitId: 'adunit-xxxxxx'  // 微信后台申请
    })

    // 2. 监听 web-view 消息
    // web-view 通过 postMessage 发消息
  },

  // 接收 H5 的广告请求
  handleMessage(e) {
    const data = e.detail.data[0]
    if (data.type === 'show_ad' && data.adType === 'rewarded') {
      this.showRewarded(data.scene)
    }
  },

  showRewarded(scene) {
    this.rewardedAd.show()
      .then(() => {
        // 完成后通知 H5
        // 通过修改 web-view src 的 hash 或 postMessage
      })
      .catch(() => {
        // 广告加载失败 → 直接给奖励（体验优先）
      })
  }
})
```

### 4.2 广告单元 ID 申请

| 广告位 | adUnitId | 微信后台路径 |
|--------|----------|-------------|
| 激励视频 | 待申请 | 小程序后台 → 流量主 → 广告管理 → 激励视频 |
| Banner | 待申请 | 同上 → Banner 广告 |
| 插屏 | 待申请 | 同上 → 插屏广告 |

## 五、浏览器开发模式

本地开发时微信 API 不可用，用以下方式模拟：

```
激励视频 → confirm('观看 15 秒广告？奖励：xxx')
Banner   → console.log + 底部模拟横幅
插屏     → 跳过（开发时不打扰）
```

代码里 `AdManager` 自动检测环境，浏览器下走 Mock 模式。
